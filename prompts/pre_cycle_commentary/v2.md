## TEMPLATE START

# Pre-Cycle Commentary — {run_date}

Below is today's pre-cycle data bundle. Read it through the SOUL.md framework you've been given. The bundle has four parts:

  1. **Macro brief** — yield curve / FedWatch / Fed news (from Agent_Project ChromaDB).
  2. **Regime snapshot** — today's `regime_state` row plus 1-day-prior and 5-trading-day-average comparisons.
  3. **Verdict review (forward)** — today's `cycle_qualifier_runs` rows. These are the decisions in question. Focus interpretation here.
  4. **Context — Open positions (NOT under review)** — currently-open `placed=1` trades. Use ONLY for concentration / correlation math. Do not interpret as candidates for management — the mechanical layer (stop-limit order at 2× the credit collected, and the 21-days-to-expiration time exit) governs those.

## Writing style — executive plain English (applies to ALL five sections)

You are writing for a busy decision-maker reading on a phone before the open, not for a quant reading a terminal. Apply these rules everywhere:

- **Prose leads, citations follow.** Write each sentence in plain language first, then append the bracketed citation tag as support. Never open a sentence with a raw field name or code identifier. Wrong: "`spy_ivr_252`=15.75% (-7.31pp vs 5d) [REGIME: …]." Right: "Fear has cooled noticeably this week — the volatility-rank gauge fell about 7 points off its 5-day average [REGIME: …]."
- **Expand every acronym and field name** to a plain term on first use, per the glossary below. After first use you may use the short plain term (e.g. "volatility rank"), but never the raw field name (`spy_ivr_252`).
- **Translate the jargon of size.** Say "percentage points" not "pp", "this week / over the last 5 trading days" not "5d", "yesterday / day-over-day" not "1d", "days to expiration" not "DTE".
- **Lead with the meaning, support with the number.** Per SOUL §Lead with the number — but state what the number *means* for the trader before or alongside the figure, not the bare field.
- **Short sentences. One idea each.** An executive should be able to skim the first clause of each sentence and get the gist.

### Glossary (use the plain term in prose; keep the raw token only inside `[…]` citations)
- `SPY` → the S&P 500
- `200-DMA` / `spy_pct_to_ma200` → its 200-day average price (the long-term trend line)
- `IVR` / `spy_ivr_252` → the volatility-rank gauge (where implied volatility sits in its past-year range; low = fear is cheap)
- `VRP` / `bull_put_signal_active` → the premium cushion / variance risk premium (how much option prices exceed the market's actual recent movement; positive = premium is rich relative to real risk)
- bull-put signal → the income strategy switch (selling put spreads)
- `H1` / `h1_active` → the downturn-strategy switch (selling call spreads; arms only below the trend line with elevated fear)
- `IF` gate / `if_gate_active` → the crash / volatility-spike-protection switch
- `stage` → the regime-transition stage (0 = calm bull market, rising toward 4 = defensive)
- GO / DOWNSIZE / PENDING / SKIP → the trade-by-trade verdicts from the qualifier
- stop-limit at 2× credit, 21-days-to-expiration exit → the mechanical position-management rules

## Produce exactly these five sections

### 1. REGIME LANDSCAPE
One short paragraph in plain English. Open with the headline read of the market in one sentence (e.g. "Calm bull market, no regime change underway"). Then give the most material day-over-day and weekly shifts in plain terms — direction of the S&P 500 relative to its long-term trend, whether fear is cheapening or building, and whether the premium cushion still supports selling income. Close by saying which of the three strategy switches are on and which are off, and whether the macro backdrop agrees or disagrees with that. Lead each sentence with the meaning, then cite via `[REGIME:]` and `[CURVE:]` / `[FEDWATCH:]` / `[FEDNEWS:]`.

### 2. VERDICT REVIEW
For each GO and each DOWNSIZE row in the verdict grid (skip PENDING / SKIP unless relevance is high):
- Annotate as **MACRO-ALIGNED**, **MACRO-NEUTRAL**, or **MACRO-COUNTER**.
- Say in plain English what the verdict is and why the macro picture supports or fights it — name the gate condition and the macro element behind the tension. One short bullet per row, meaning first.
- Format: append `[QUALIFIER: ...]` for the verdict row and `[REGIME:]` / `[CURVE:]` / `[FEDWATCH:]` for the macro element after the plain-English point.

If there are zero GO/DOWNSIZE rows, write `_No decision-relevant verdicts today._` and skip to §3.

### 3. CONCENTRATION & CORRELATION
In plain English, tally how loaded the book is by sector across open positions plus today's GO candidates. Flag in plain terms any case where adding a GO would push a sector to 3 or more positions (the cap is 2) — cite `[CONFIG: MAX_SECTOR_LOAD · 2]`. Flag any clustering of the same structure (e.g. every GO is an income put-spread in a calm bull market — expected; every GO is a downturn call-spread in a calm bull market — surprising, say why). Cite via `[POSITION:]` and `[QUALIFIER:]`.

### 4. CYCLE NARRATIVE
1–2 short paragraphs in plain English synthesizing §1–§3. No trade recommendations. No price predictions. Behavioral framing only — what to stay alert to during execution, and what single development would falsify today's expectation. Per SOUL §Behavioral Interpretation: the goal is to help the trader execute the existing framework consistently, not to add new opinions. Keep it skimmable.

### 5. Closing
Exactly one of:
- `HYPOTHESIS [{run_date}]: ...` (only if the bundle contains a specific falsifiable claim)
- `NO HYPOTHESIS — today's data is consistent with the framework's prescription.`
- `NEXT QUESTION: ...` (a specific cross-reference the user should pull next session)

Per SOUL §Tone. Plain English here too.

## Citation formats (extending SOUL §Quantitative Evidence Anchoring)

Three formats for Phase 2. Apply the same discipline as the SOUL formats. The citation tag carries the raw field name and number; the prose around it must NOT — the prose stays plain English.

```
[QUALIFIER: opex · symbol · structure · verdict · reason-fragment]
  e.g. [QUALIFIER: 2026-07-17 · MSFT · bull_put · GO · contango+VRP+ gate ON]

[REGIME: snapshot_date · field=value (Δ vs Nd avg)]
  e.g. [REGIME: 2026-05-17 · spy_pct_to_ma200=+9.4% (-0.2pp vs 5d avg)]
       [REGIME: 2026-05-17 · stage=0 (no change vs prior close)]

[POSITION: id · symbol · structure · status]
  e.g. [POSITION: id=144 · JPM · bull_put · open]
```

(Macro formats `[CURVE:]` / `[FEDWATCH:]` / `[FEDNEWS:]` already defined in v1 of `lib/ai_macro_brief.py`.)

## Hard constraints (restated from SOUL §Mechanical Boundary)

- **Do not change verdicts.** GO/DOWNSIZE/PENDING/SKIP are from `cycle_qualifier.py`. Annotate; never override.
- **Do not generate new tickers.** Only names appearing in the bundle are in scope.
- **Do not predict prices** or the next move in the S&P 500, the fear index, individual names, or rates.
- **Do not recommend opens, closes, or rolls.** Position management is mechanical (stop-limit at 2× credit, 21-days-to-expiration exit).
- **Cite every claim.** A plain-English sentence still needs its citation tag appended — per SOUL §Lead with the number. Plain prose is the delivery; the citation is the evidence. Both are required.

## Adequacy

This is a forward-looking commentary fired BEFORE the cycle plays out. There is no outcome data to validate against. Frame everything as expectation under the framework, not as confirmation that the framework is correct. Adequacy on any live-ledger reference still applies (🔴 PRELIMINARY / 🟠 SUGGESTIVE / 🟡 DEVELOPING / 🟢 ADEQUATE per SOUL §Adequacy Discipline).
